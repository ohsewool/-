# ModelMate 발표 준비 런북

## 기준

- 발표일: 2026-06-08 월요일
- 현재 목표: 11월 상용화 로드맵은 잠시 멈추고, 발표 당일 안정적으로 보여주는 데 집중한다.
- 발표 메시지: ModelMate는 임의 CSV를 올리면 데이터 성격, 맞힐 값, 제외할 정보, 모델 후보, 위험 요소, 다음 행동을 정리해 주는 범용 CSV AutoML 서비스다.

## 발표 전 최우선 작업

1. Railway 배포가 최신 GitHub `main` 기준으로 정상인지 확인한다.
2. 발표용 메인 데이터셋 1개와 백업 데이터셋 1개를 정한다.
3. 업로드, 자동 판단, 모델 비교, 에이전트 결론, 보고서, 이유 보기, 작업 기록 흐름을 한 번에 시연한다.
4. 실패 가능성이 있는 새 기능 추가는 중단하고, 문구/동선/오류 처리만 고친다.
5. 발표 전에 브라우저 탭을 미리 열어두고, Railway가 느릴 때를 대비해 저장된 결과 화면도 준비한다.

## 5~7분 데모 순서

1. 홈 화면에서 ModelMate가 무엇을 하는 서비스인지 한 문장으로 설명한다.
2. `/upload`에서 CSV를 올리고, 데이터 종류와 맞힐 값 판단을 보여준다.
3. `/model-lab`에서 여러 모델 점수를 비교하고 최고/최저 모델 색상 차이를 설명한다.
4. `/agent`에서 AI 에이전트가 데이터 판단, 모델 선택 이유, 위험 요소, 다음 행동을 정리하는 부분을 보여준다.
5. `/report`와 `/xai`에서 비전공자용 결과 요약과 이유 보기를 보여준다.
6. `/history`에서 이전 실험을 다시 열 수 있음을 보여준다.
7. 시간이 남으면 `/predict` 또는 `/deploy`에서 새 데이터 예측/API 공유 흐름을 짧게 보여준다.

## 발표용 핵심 문장

- "CSV만 넣으면 어떤 예측 문제로 만들 수 있는지 ModelMate가 먼저 정리합니다."
- "모델을 하나로 고정하지 않고 데이터셋에 맞는 후보를 비교합니다."
- "AI 에이전트는 결과만 말하는 것이 아니라 왜 이 타겟과 모델을 선택했는지, 발표자가 무엇을 말하면 되는지까지 정리합니다."
- "작업 기록과 저장 모델 흐름이 있어서 단순 데모가 아니라 서비스형 AutoML 구조로 확장할 수 있습니다."

## 데이터셋 운영

- 메인 데이터셋: 업로드와 모델 비교가 이미 안정적으로 통과한 CSV를 사용한다.
- 백업 데이터셋: Pima 또는 공공교통/이용자 통계류 CSV처럼 도메인 판단이 잘 보이는 파일을 준비한다.
- 실패 사례: ID, 이름, 주소 위주의 목록형 CSV는 예측 데이터가 아니라고 거부되는 것을 필요할 때만 보여준다.
- 발표 당일에는 처음 보는 대용량 파일, 인코딩이 애매한 파일, 텍스트 문서형 CSV는 사용하지 않는다.

## 장애 대응

- 업로드가 느리면: 이미 저장된 작업 기록에서 결과를 연다.
- 에이전트 응답이 늦으면: 보고서와 이유 보기 화면으로 넘어간다.
- Railway가 느리면: 미리 열어둔 탭을 사용하고, 새로고침은 최소화한다.
- 모델 학습이 실패하면: 백업 데이터셋으로 즉시 전환한다.

## 발표 전 금지 사항

- 큰 구조 변경.
- 인증/권한/결제/API 보안 같은 무거운 기능 신규 구현.
- 발표 당일 검증되지 않은 새 데이터셋 사용.
- 설명 문구를 과하게 늘리는 UI 수정.

## 발표 후 다시 이어갈 11월 방향

- 계정별 영구 저장, 실험 재사용, 저장 모델 버전 관리, 실제 API 인증, 공유 권한, 장기 QA를 다시 우선순위로 둔다.
