표지

약 30초
화면 포인트

프로젝트명, 팀원, 졸업 프로젝트 정보를 짧게 소개합니다.

안녕하세요. 지금부터 저희 졸업 프로젝트인 ModelMate를 발표하겠습니다.

ModelMate는 CSV 데이터를 업로드하면 예측할 대상 설정, 모델 비교, 결과 해석, 새 데이터 예측, API 공유, 작업 기록까지 이어지는 정형 데이터 기반 AutoML 플랫폼입니다.

이번 발표에서는 사용자가 CSV를 올린 뒤 어떤 흐름으로 예측 실험을 진행하는지 중심으로 설명드리겠습니다.

다음: 프로젝트 배경
2

CSV 데이터와 예측 목적 사이의 간극

약 60초
핵심 메시지

CSV는 쉽게 구할 수 있지만, 예측 문제로 바꾸는 과정이 어렵다는 점을 강조합니다.

처음 프로젝트는 특정 사고 예측 프로젝트로 출발했습니다. 하지만 특정 데이터에만 의존하면 데이터셋이 바뀔 때마다 전처리와 모델 구성이 크게 달라지고, 다른 분야로 확장하기 어렵다는 한계가 있었습니다.

그래서 방향을 바꾸어 사고 데이터뿐 아니라 건강 데이터, 공공데이터, 업무 데이터처럼 다양한 CSV를 예측 실험으로 바꿀 수 있는 범용 AutoML 서비스로 확장했습니다.

CSV 데이터는 쉽게 확보할 수 있지만, 실제로 예측 문제로 바꾸는 과정은 어렵습니다. 어떤 컬럼을 맞힐지, 이 문제가 분류인지 회귀인지, 어떤 모델이 적합한지, 결과를 어떻게 해석해야 하는지를 사용자가 직접 판단해야 하기 때문입니다.

ModelMate는 이 판단 과정을 서비스 안에서 정리해주는 것을 목표로 합니다.

다음: 전체 접근 방식
3

CSV 기반 예측 실험 자동화

약 60초
설명 순서

CSV 업로드 → 분석 설정 → 모델 비교 → 결과 설명 → 결과 재사용

전체 흐름은 다섯 단계입니다.

첫 번째는 CSV 업로드입니다. 사용자가 파일을 올리면 데이터 구조와 기본 품질을 확인합니다.

두 번째는 분석 설정입니다. 업로드된 데이터에서 예측할 만한 값과 제외해야 할 컬럼을 정리합니다.

세 번째는 모델 비교입니다. 여러 모델을 같은 기준에서 교차검증으로 비교해 리더보드를 만듭니다.

네 번째는 결과 설명입니다. 단순 점수만 보여주는 것이 아니라 어떤 정보가 예측에 중요했는지 함께 보여줍니다.

마지막은 결과 재사용입니다. 학습된 모델로 새 데이터를 예측하거나 API 공유, 작업 기록으로 이어질 수 있습니다.

다음: 구현 구조
4

React/Vite · FastAPI 기반 웹 서비스 구현

약 50초
기술 포인트

프론트엔드, 백엔드, AutoML, 실험 기록 모듈을 연결해서 설명합니다.

구현은 React/Vite 기반 프론트엔드와 FastAPI 기반 백엔드로 구성했습니다.

프론트엔드에서는 업로드, 모델 선택, 결과 요약, XAI, AI 분석 코치 화면을 구성했습니다.

백엔드에서는 CSV 처리, 데이터 검증, 타겟 추천, 모델 학습과 비교 API를 구현했습니다.

AutoML 모듈은 분류와 회귀를 판별하고 여러 모델을 교차검증으로 비교합니다. 또한 실험 기록 모듈을 통해 로그인 사용자별 분석 기록과 결과 재확인 흐름을 제공합니다.

다음: 핵심 서비스 화면
5

핵심 서비스 화면

약 70초
화면 3개만 설명

CSV 업로드, 모델 비교, 결과 요약만 중심으로 설명합니다.

서비스 화면은 핵심 흐름만 보여드리겠습니다.

먼저 CSV 업로드 화면입니다. 파일을 올리면 행 수, 컬럼 수, 결측값, 데이터 종류, 맞힐 값 후보를 정리합니다. 이 단계에서 모델이 무엇을 예측하고 어떤 정보를 사용할지 준비합니다.

다음은 모델 비교 화면입니다. Logistic Regression, Random Forest, Gradient Boosting, LightGBM, XGBoost, Decision Tree 등 여러 모델을 같은 기준으로 비교합니다. 그래프에서는 최고 성능 모델과 낮은 성능 모델을 구분해 볼 수 있습니다.

마지막은 결과 요약 화면입니다. 선택된 모델, 성능 지표, 중요한 변수, 결과 해석을 한 화면에서 확인할 수 있습니다. 사용자는 단순히 점수만 보는 것이 아니라 왜 이 모델이 선택되었는지까지 이해할 수 있습니다.

다음: AI 분석 흐름
6

분석 문제 정리와 다음 행동 추천

약 60초
Agentic 포인트

AI가 결과만 내는 것이 아니라, 분석 준비와 다음 행동까지 안내한다는 점을 말합니다.

ModelMate에서 AI 분석 코치는 사용자가 직접 판단하기 어려운 분석 준비 과정을 보조합니다.

업로드된 CSV를 보고 데이터 분야를 판단하고, 예측 목적과 타겟 후보를 설명합니다. 또한 ID, 날짜, 정답을 직접 알려주는 컬럼처럼 예측을 방해할 수 있는 컬럼은 제외 후보로 분리합니다.

이후 바로 모델 비교를 진행해도 되는지, 데이터를 다시 확인해야 하는지 다음 행동으로 연결합니다.

즉, AI가 분석을 끝냈다고 말하는 것에서 멈추지 않고, 사용자가 다음에 무엇을 해야 하는지까지 안내하는 흐름입니다.

대표 데이터 출처 기반 검증 결과

약 50초
검증 범위

UCI, 데이콘, 공공기관, 비정상 파일 테스트를 짧게 언급합니다.

검증은 여러 종류의 CSV를 기준으로 진행했습니다.

UCI 공개 데이터셋으로는 당뇨, 품질, 분류/회귀 예제처럼 머신러닝 학습에 자주 쓰이는 정형 데이터에서 기본 AutoML 흐름을 확인했습니다.

데이콘 데이터셋에서는 수면이나 건강 라이프로그처럼 컬럼명이 축약된 데이터에서 타겟 의미를 사람이 이해하는 말로 바꾸는지 확인했습니다.

공공기관 데이터에서는 한글 컬럼과 행정 데이터가 포함된 CSV에서 도메인 추론과 인코딩 처리를 확인했습니다. 또한 잡담성 텍스트나 너무 작은 파일처럼 예측 대상이 불명확한 파일은 안내 메시지로 처리하도록 개선했습니다.

범용 AutoML 서비스로의 확장 방향

약 50초
미래 방향

11월 최종 졸업프로젝트까지 가져갈 확장 방향입니다.

향후 방향은 세 가지입니다.

첫 번째는 데이터 품질 진단 강화입니다. 누수 컬럼, ID성 컬럼, 날짜 컬럼, 비예측성 파일을 더 정교하게 판별할 계획입니다.

두 번째는 모델 관리 강화입니다. 저장 모델, 버전 기록, 재학습, 새 데이터 예측 흐름을 실제 서비스처럼 확장할 수 있습니다.

세 번째는 SaaS 기능 강화입니다. 사용자별 작업 공간, 공유 링크, API 키, 관리자 화면을 고도화해 실제 상용 서비스에 가까운 구조로 발전시키는 것이 목표입니다.
