Problem Definition

ModelMate의 문제 정의

비전공자 또는 초급 사용자는 CSV 데이터를 가지고 있어도 예측 대상, 모델 선택, 결과 신뢰 근거를 스스로 정하기 어렵다.

예측 대상

어떤 컬럼을 target으로 삼아야 할지 모른다. 예를 들어 Outcome, S3 같은 컬럼명이 실제로 무엇을 의미하는지 해석이 필요하다.

모델 선택

Random Forest, Gradient Boosting, LightGBM 계열처럼 모델마다 강점이 달라 교차검증 기반 비교가 필요하다.

결과 해석

성능 점수만으로는 부족하다. Feature Importance, SHAP 같은 XAI 정보가 있어야 결과를 설명할 수 있다.

Approach

예측 실험을 완성하는 5단계 AutoML 흐름

1

CSV 업로드

파일과 데이터 구조를 확인

2

타겟 추천

예측할 값을 후보로 제안

3

모델 비교

교차검증 기반 리더보드 생성

4

결과 설명

XAI와 변수 중요도로 이유 제공

5

재사용

새 데이터 예측과 기록 저장

이 흐름은 제안서의 자동 모델링 모듈, 설명가능성 분석 모듈, 리포트 생성 모듈, 실험 로그 및 재현 모듈을 웹 서비스 화면으로 연결한 것이다.

Implementation

React/Vite · FastAPI 기반 웹 서비스 구현

발표 후 실제 사이트에서 CSV 업로드부터 결과 확인까지 시연

Frontend

React/Vite 기반으로 업로드, 모델 선택, 결과 요약, XAI, AI 분석 코치 화면을 구성했다.

Backend

FastAPI 기반으로 CSV 처리, 데이터 검증, 타겟 추천, 모델 학습/비교, Optuna 하이퍼파라미터 튜닝을 수행한다.

Deploy

GitHub main 브랜치와 Railway production 배포를 연결하고, 데모 모드로 API 토큰 소모 없이 테스트한다.
