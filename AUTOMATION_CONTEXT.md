# ModelMate 자동화 기준 보존

## 보존된 11월 자동화 방향

11월 최종 졸업 프로젝트 목표는 ModelMate를 고정 샘플 데모가 아니라 상용화 가능한 범용 CSV AutoML SaaS처럼 만드는 것이다.

핵심 방향은 다음과 같다.

1. 임의 CSV 업로드 시 데이터 분야, 예측 목적, 맞힐 값, 제외 컬럼, 모델 선택, 위험 요소, 다음 행동을 자동 판단한다.
2. 계정/작업공간 가치를 강화해 사용자별 저장 데이터셋, 실험 기록, 보고서, 저장 모델을 다시 열 수 있게 한다.
3. 실험 재사용을 강화해 저장된 타겟, 제외 컬럼, 선택 모델, 점수, 보고서를 복원한다.
4. 모델 관리를 강화해 이름 있는 저장 모델, 버전 형태 기록, 최고 모델 재사용, 새 데이터 예측을 연결한다.
5. 공유/API 흐름을 강화해 공유 링크, 공개/비공개, API 키 형태 UX, 사용 예시를 제공한다.
6. 임의 CSV 대응력을 높이기 위해 공공, 건강, 제조, 교통, 시설/안전, 금융, HR/고객, invalid, tiny, text-like, malformed, non-predictive CSV를 QA한다.
7. UI는 긴 설명을 줄이고 보조 정보는 사이드 패널로 분리한다.

## 현재 임시 자동화 방향

발표일이 2026-06-08 월요일로 가까워졌기 때문에 11월 로드맵은 일시 중단한다.

현재 자동화는 발표 안정화에만 집중한다.

1. GitHub `main`, 작업트리, Railway URL, 최신 배포 반영 여부를 먼저 확인한다.
2. 발표용 메인 데이터셋과 백업 데이터셋으로 업로드, 모델 비교, 에이전트, 보고서, 이유 보기, 작업 기록 흐름을 점검한다.
3. 발표자가 말할 수 있는 핵심 문장과 장애 대응 플랜을 유지한다.
4. 새 기능 추가는 최소화하고, 발표를 망칠 수 있는 오류, 문구, 동선 문제만 수정한다.
5. 안정화된 변경만 빌드, no-token 점검, 커밋, 푸시, Railway 배포 확인까지 진행한다.

## 전환 기준

- 발표 전: 발표 안정화 모드 유지.
- 발표 후: 11월 상용화 로드맵으로 복귀.
