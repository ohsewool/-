# ModelMate Demo Dataset Playbook

## 목적

11월 발표에서는 데이터셋을 많이 보여주는 것보다, 대표 사례를 안정적으로 반복 시연하는 것이 더 중요하다. 아래 3개를 기본 데모 후보로 고정한다.

## 대표 데이터셋

| 우선순위 | 파일 | 발표 역할 | 보여줄 메시지 |
| --- | --- | --- | --- |
| 1 | `tmp_datasets/pima.csv` | 메인 데모 | 의료/건강 CSV를 넣으면 당뇨 여부 예측 모델을 만들 수 있다 |
| 2 | `tmp_datasets/ch2025.csv` | 보조 데모 | 수면/건강 라이프로그도 분류 문제로 처리할 수 있다 |
| 3 | `tmp_datasets/seoul_bike.csv` | 확장 데모 | 공공 데이터의 숫자 예측도 처리할 수 있다 |

`tmp_datasets/playground.csv`는 공공시설 데이터 대응력을 보여주기 좋지만 파일이 크고 화면 설명이 길어질 수 있어, 시간이 남을 때만 사용한다.

## 메인 데모 순서

1. 데모 모드를 켠다.
2. 발표자용 안내가 필요하면 `/upload?demo=1` 또는 `/upload?presenter=1`로 들어간다.
3. `pima.csv`를 업로드한다.
4. AI가 데이터 분야와 맞힐 값을 어떻게 판단했는지 짧게 설명한다.
5. 모델 비교를 실행한다.
6. 결과 요약에서 선택 모델과 성능을 보여준다.
7. 이유 보기에서 중요한 정보를 보여준다.
8. 시간이 남으면 작업공간에서 데이터셋과 실험 기록이 연결된 것을 보여준다.
9. 더 시간이 남으면 모델 공유 화면에서 API URL이 만들어지는 흐름을 보여준다.

## 발표 멘트

이 서비스는 CSV를 넣으면 데이터의 성격을 판단하고, 예측할 값을 고른 뒤, 여러 모델을 비교해 가장 적합한 모델과 이유를 보여줍니다.

## 실패 대비

- 업로드가 실패하면 파일 형식 검증 메시지를 보여주며 "잘못된 CSV도 막는다"고 설명한다.
- AI 설명이 느리면 데모 모드로 전환해 저장된 흐름 중심으로 시연한다.
- 모델 비교가 오래 걸리면 Pima 또는 Seoul bike처럼 이미 검증된 작은 데이터셋으로 바꾼다.
- 브라우저가 이전 번들을 보고 있으면 `Ctrl + F5`로 새로고침한다.

## 발표 전 루틴

1. `python scripts/run_final_qa.py`
2. Railway URL 접속
3. 데모 모드 켜기
4. `pima.csv` 업로드 리허설
5. `/history`, `/report`, `/xai`, `/deploy` 화면 확인
