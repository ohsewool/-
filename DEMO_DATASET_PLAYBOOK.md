# ModelMate Demo Playbook

## 목적

11월 발표의 기본 전략은 고정된 샘플만 보여주는 것이 아니라, 즉석 CSV도 처리할 수 있는 범용 AutoML 흐름을 보여주는 것이다. 검증된 데이터셋은 발표 중 문제가 생겼을 때 사용하는 백업 카드로 둔다.

## 기본 시연 원칙

- 처음 보는 CSV를 넣어도 데이터 분야, 맞힐 값, 제외할 정보를 자동으로 판단하는 모습을 보여준다.
- 발표자는 결과가 완벽히 맞는다고 말하기보다, "AI가 먼저 판단하고 사람이 수정할 수 있다"고 설명한다.
- 모델 비교와 이유 보기는 반드시 보여준다.
- 새 데이터 예측과 API 공유는 시간이 남을 때만 보여준다.

## 즉석 CSV 시연 순서

1. CSV 파일을 업로드한다.
2. 데이터 분야와 맞힐 값 추천을 확인한다.
3. 추천이 이상하면 사람이 직접 맞힐 값을 바꿀 수 있음을 보여준다.
4. 분석 준비를 실행한다.
5. 모델 비교를 실행한다.
6. 결과 요약과 이유 보기를 보여준다.
7. 작업공간에서 데이터셋과 실험 기록이 이어진다는 점을 보여준다.

## 백업 데이터셋

| 용도 | 파일 | 쓰는 상황 |
| --- | --- | --- |
| 의료/건강 분류 | `tmp_datasets/pima.csv` | 즉석 파일이 없거나 발표 시간이 짧을 때 |
| 수면/건강 분류 | `tmp_datasets/ch2025.csv` | 프로젝트 초반 문제 정의를 보여줄 때 |
| 공공 데이터 회귀 | `tmp_datasets/seoul_bike.csv` | 숫자 예측도 가능하다는 점을 보여줄 때 |
| 공공시설 분류 | `tmp_datasets/playground.csv` | 공공기관 데이터 대응력을 보조로 보여줄 때 |

## 실패 대비 멘트

- "이 파일은 예측 모델용 표 데이터로 보기 어려워서 업로드 단계에서 막았습니다."
- "AI가 추천한 맞힐 값은 자동 판단이므로, 사용자가 직접 수정할 수 있습니다."
- "도메인을 확신하지 못하는 데이터는 도메인 확인 필요로 표시해 과도한 추정을 피합니다."
- "발표 중 학습이 오래 걸릴 경우 검증된 백업 데이터셋으로 같은 흐름을 보여드리겠습니다."

## 발표 전 루틴

1. `python scripts/run_final_qa.py`
2. Railway URL 접속
3. 일반 `/upload`에서 발표자용 안내가 노출되지 않는지 확인
4. 즉석 CSV 1개 또는 백업 CSV 1개 업로드
5. `/history`, `/report`, `/xai`, `/deploy` 화면 확인
