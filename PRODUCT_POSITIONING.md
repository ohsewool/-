# Product Positioning

## 한 줄 정의

ModelMate는 비전공자를 위한 CSV 기반 AutoML 데모 서비스다.

## 비교 기준

| 서비스 | 강점 | ModelMate와 차이 |
| --- | --- | --- |
| Google Vertex AI | 기업용 AutoML, 강력한 클라우드 기능 | 진입 장벽이 높고 설정이 복잡하다 |
| DataRobot | 모델 관리, MLOps, 대규모 운영 | 졸업프로젝트 범위를 넘어서는 기업용 기능이 많다 |
| Obviously AI | 비전공자 친화적 예측 분석 | ModelMate가 지향하는 사용성 방향과 가장 가깝다 |
| H2O Driverless AI | 강력한 AutoML과 설명 기능 | 전문 사용자와 기업 환경에 더 가깝다 |

## 우리 프로젝트의 포지션

ModelMate는 대형 AutoML 서비스처럼 모든 운영 기능을 제공하는 것이 목표가 아니다.

목표는 다음과 같다.

- CSV 파일만으로 시작할 수 있다.
- 모델 선택 과정을 자동화한다.
- 결과를 비전공자도 이해할 수 있게 요약한다.
- 예측 근거를 함께 보여준다.
- 실험 기록과 저장 모델을 작업공간에서 이어서 관리한다.

## 발표에서 강조할 차별점

- 복잡한 AI 용어보다 사용 흐름을 먼저 보여준다.
- 모델 성능뿐 아니라 예측 근거도 제공한다.
- 기능을 데이터 넣기, 자동 분석, 모델 고르기, 결과 요약, 이유 보기 흐름으로 단순화했다.
- 예측과 API 공유는 선택 확장 기능으로 제공한다.
- 데이터셋-실험-저장 모델 흐름을 자동 QA로 검증한다.
- 고정 샘플이 아니라 즉석 CSV도 처리하는 범용성을 보여준다.

## 11월까지 강화할 제품성

- 데모 신뢰도: 즉석 CSV를 기본으로 보여주고, 검증 데이터셋은 백업으로만 사용한다.
- 설명력: 타겟 선택, 모델 선택, 전처리, 성능 개선 판단이 비전공자에게 자연스러워야 한다.
- 작업공간 가치: 로그인 후 데이터셋, 실험, 저장 모델이 이어져 보이는 이유가 명확해야 한다.
- 운영 안정성: 이상한 파일, 빈 데이터, 도메인 미확인 데이터도 친절하게 처리해야 한다.

## 상업성을 높이는 방향

- 계정 가치: 로그인하면 데이터셋, 실험, 저장 모델이 내 작업공간에 누적되어야 한다.
- 재사용 가치: 저장 모델을 다시 테스트하고 공유 URL로 사용할 수 있어야 한다.
- 신뢰 가치: 데이터 검증, 자동 QA, 모델 버전, 저장 상태를 보여줘야 한다.
- 협업 가치: 관리자 화면과 실험 비교로 팀 프로젝트처럼 보이게 해야 한다.
- 전환 가치: 무료 데모는 쉽게 시작하고, 고급 기능은 작업공간/모델 관리/API 공유로 이어져야 한다.
