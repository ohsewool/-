# ModelMate Feedback Collection Guide

This guide is for small beta sessions with 5-15 users. Use GitHub issues,
interview notes, or a Google Form/Typeform placeholder. Do not collect sensitive
CSV files unless the user has permission and the data is safe to share.

## Suggested Collection Channels

- GitHub issue: use `.github/ISSUE_TEMPLATE/beta_feedback.md`
- Interview notes
- Google Form placeholder: `<add form link before beta>`
- Typeform placeholder: `<add form link before beta>`

## English Questions

1. What were you trying to predict?
2. Was it clear which CSV file to upload?
3. Did you understand the target recommendation?
4. Did the warnings help you trust or question the result?
5. Was the report useful?
6. Would you reuse the prediction API?
7. What part was confusing?
8. What would make you pay for this?
9. What feature felt unnecessary?
10. Would you prefer Free / Pro / Team pricing?
11. Did you feel confident enough to save or share the report?
12. What should ModelMate explain better before training starts?

## Korean Questions

1. 무엇을 예측하려고 했나요?
2. 어떤 CSV를 올려야 하는지 명확했나요?
3. 추천된 맞힐 값(target)을 이해했나요?
4. 경고 문구가 결과를 신뢰하거나 의심하는 데 도움이 되었나요?
5. 보고서가 저장하거나 공유할 만큼 유용했나요?
6. 예측 API를 다시 사용할 것 같나요?
7. 가장 헷갈린 부분은 무엇이었나요?
8. 어떤 기능이 있으면 비용을 지불할 것 같나요?
9. 불필요하게 느껴진 기능은 무엇인가요?
10. Free / Pro / Team 중 어떤 가격 구조가 가장 자연스럽나요?
11. 결과의 한계와 주의사항이 충분히 보였나요?
12. 학습 전에 ModelMate가 더 설명해야 하는 부분은 무엇인가요?

## Follow-Up Notes

After each session, record:

- whether the user completed the demo flow;
- where they stopped;
- whether they understood the report limitations;
- whether they would use their own CSV next time;
- one change that would most improve trust.
