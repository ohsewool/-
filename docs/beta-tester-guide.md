# ModelMate Beta Tester Guide

Thank you for trying ModelMate. This beta should take about 5-10 minutes.

## Who This Test Is For

ModelMate is especially useful to test if you are:

- a student;
- a non-technical user with CSV files;
- a small startup operator;
- a marketing, sales, or RevOps user;
- a small business owner;
- a freelance analyst or consultant.

## What CSV To Prepare

A good CSV has:

- one row per customer, event, product, case, or time period;
- several columns known before prediction;
- one column you might want to predict;
- no sensitive personal, payment, medical, or regulated information;
- enough rows for a small model comparison.

Avoid uploading CSV files that are only ID lists, names, addresses, passwords,
API keys, or free-form text notes.

## Start With Sample Data

If you do not have a safe CSV ready, start with the sample cards on the upload
screen:

- Customer churn demo
- Manufacturing quality demo
- Public bike signup demo

Download one sample CSV, upload it, and follow the guided flow.

## Test Steps

1. Open the ModelMate app.
2. Read the landing page message.
3. Go to the upload screen.
4. Upload your CSV or a sample CSV.
5. Check the data judgment and recommended target.
6. Run model comparison.
7. Open the result/report screen.
8. Review analysis trace, trust panel, evidence summary, and limitations.
9. Try report export.
10. Check prediction API documentation or project history/rerun.

## How To Leave Feedback

Use one of these paths:

- GitHub issue templates in `.github/ISSUE_TEMPLATE/`
- Feedback form placeholder: `<add Google Form or Typeform link>`
- Direct interview notes

## What To Include When Reporting A Bug

- What happened
- What you expected
- Steps to reproduce
- CSV/sample dataset used
- Screenshot or log
- Browser and OS
- Whether the issue blocks the demo

## Safety Reminder

Do not upload sensitive personal information, payment data, medical data,
private customer lists, passwords, API keys, or regulated records during beta
testing.
