# ModelMate November Roadmap

## 현재 판단

11월 졸업프로젝트 발표 기준으로 ModelMate는 핵심 기능 구현을 지나 안정화와 제품 완성도 단계에 들어섰다. 앞으로는 기능을 무작정 늘리기보다, 처음 보는 사람이 빠르게 이해하고 발표 중 끊기지 않게 만드는 것이 우선이다.

## 진행 단계

| 순서 | 목표 | 상태 |
| --- | --- | --- |
| 1 | CSV 업로드, 타겟 선택, 모델 비교 | 완료 |
| 2 | 결과 요약, 이유 보기, 보고서 | 완료 |
| 3 | 로그인, 작업공간, 관리자 | 완료 |
| 4 | 데이터셋-실험-저장 모델 연결 | 완료 |
| 5 | 자동 QA와 배포 검증 | 진행 중 |
| 6 | 대표 데모 시나리오 고정 | 다음 우선순위 |
| 7 | UI/UX 상용화 polish | 계속 진행 |
| 8 | 최종 발표 리허설과 문서 마감 | 발표 전 집중 |

## 다음 고도화 우선순위

1. 대표 데모 데이터셋을 2~3개로 고정한다.
2. 데모 모드에서 업로드부터 보고서까지 토큰 없이 재현 가능한 흐름을 더 선명하게 만든다.
3. AI 에이전트가 데이터 판단, 모델 선택 이유, 위험 노트, 다음 행동을 한 화면에서 설명하게 다듬는다.
4. 작업공간에서 데이터셋, 실험, 저장 모델의 관계를 더 직관적으로 보여준다.
5. 모바일/작은 화면에서 버튼, 표, 카드가 겹치지 않도록 계속 점검한다.
6. `scripts/run_final_qa.py`를 발표 전 기본 점검 루틴으로 유지한다.

## 발표에서 보여줄 핵심 문장

ModelMate는 CSV를 넣으면 데이터 성격을 판단하고, 예측할 값을 고른 뒤, 여러 모델을 비교해 결과와 이유까지 보여주는 비전공자 친화 AutoML 서비스다.

## 리스크

- 학습 벤치마크는 시간이 오래 걸릴 수 있어 캐시 통과가 사용된다.
- 외부 AI 토큰이 부족할 수 있어 데모 모드와 저장 결과 기반 시연이 필요하다.
- 너무 많은 기능을 보여주면 핵심 메시지가 흐려질 수 있다.
- UI는 아직 대형 SaaS 수준까지는 아니므로 계속 다듬어야 한다.

## 운영 루틴

큰 변경 후에는 다음 순서로 확인한다.

1. `python scripts/run_final_qa.py`
2. 프론트 빌드
3. GitHub main 푸시
4. Railway URL 확인
5. 실제 브라우저에서 대표 데이터셋 1개 업로드 리허설
