# ModelMate 11월 프로젝트 로드맵

## 최종 목표

ModelMate를 고정 샘플 데모가 아니라, 임의 CSV를 즉석 업로드해도 데이터의 성격을 판단하고 예측 문제를 구성한 뒤 모델 비교, 이유 설명, 저장, 재사용, 공유/API까지 이어지는 범용 AutoML SaaS처럼 보이게 만든다.

## 현재 진행률

- 중간 발표 가능 상태: 80~85%.
- 11월 최종 발표 기준: 45~50%.
- 실제 상용 MVP 느낌 기준: 30~35%.
- 지금까지는 업로드, 분석, 모델 비교, 결과 설명, 에이전트 판단, 기본 작업공간, 공유/API 미리보기를 만들었다.
- 앞으로는 저장, 재사용, 관리, 안정성, QA가 핵심이다.

## 월별 기준선

### 6월: 안정 버전

- 중간 발표 가능한 안정 버전을 만든다.
- 업로드, 자동 분석, 모델 선택, 결과 요약, 이유 보기, 저장/재사용 기본 흐름을 안정화한다.
- 여러 CSV를 넣어 깨지는 부분을 줄인다.

### 7월: 작업공간/실험 관리

- 사용자별 데이터셋과 실험 기록을 관리한다.
- 과거 작업을 다시 열고 같은 설정으로 재분석할 수 있게 한다.
- 관리자 화면에서 사용자, 데이터셋, 실험, 실패 흐름을 확인한다.

### 8월: 모델 관리/재사용

- 최고 모델을 이름과 버전으로 저장한다.
- 저장 모델로 새 데이터 예측을 실행한다.
- 모델별 성능과 상태를 비교할 수 있게 한다.

### 9월: 공유/API/상용화 흐름

- 보고서 공유 링크를 정리한다.
- 공개/비공개 상태를 표시한다.
- API 키 형태의 UX와 사용 예시를 강화한다.
- 실제 운영 기능과 미리보기 기능을 명확히 구분한다.

### 10월: UI/QA/시연 완성도

- 긴 화면을 사이드 패널로 정리한다.
- 모바일/좁은 화면에서 버튼, 카드, 표가 깨지지 않게 한다.
- 공공, 의료, 제조, 교통, 시설, invalid CSV QA를 반복한다.
- 발표자가 설명하기 전에 화면 자체가 의미를 전달하게 만든다.

### 11월: 최종 발표 패키징

- 발표자료, README, 최종 보고서, 백업 데이터셋, 시연 영상, 장애 대응 시나리오를 완성한다.
- 운영 URL, GitHub, QA 결과, 발표 스토리가 서로 맞는지 최종 점검한다.

## 자동화 우선순위

1. GitHub/Railway 배포 반영을 먼저 확인한다.
2. 계정/작업공간과 실험 기록의 연속성을 강화한다.
3. 이전 실험의 target, excluded columns, selected model, score, report를 재사용하게 만든다.
4. 저장 모델, 버전, 새 데이터 예측을 강화한다.
5. 공유/API 흐름을 상용 서비스처럼 정리한다.
6. AI 에이전트의 판단 이유와 다음 행동을 강화한다.
7. 다양한 CSV에 대한 친절한 검증과 QA를 반복한다.
8. UI 과밀도를 줄이고 사이드 패널과 모바일 대응을 개선한다.
9. handoff 문서를 최신 상태로 유지한다.

## 발표 핵심 문장

ModelMate는 CSV를 넣으면 데이터의 성격을 판단하고, 무엇을 예측할지 정리한 뒤, 여러 모델을 비교해 결과와 이유까지 설명해주는 비전공자 친화 AutoML 서비스다.

## 남은 리스크

- 로그인/권한/저장소는 아직 실제 상용 보안 수준이 아니다.
- API 키 UI는 일부 미리보기이며 실제 production secret 관리와는 구분해야 한다.
- 임의 CSV 도메인 판단은 휴리스틱 기반이라 오판 가능성이 있다.
- 데이터셋이 너무 작거나 텍스트성/비정형성이 강하면 학습 품질이 낮을 수 있다.
- 기능이 많아질수록 UI가 복잡해지므로 계속 사이드 패널과 요약 중심으로 정리해야 한다.
